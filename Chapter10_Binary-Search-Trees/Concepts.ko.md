## 1. 이진 탐색 트리 정의

### 1.1 정의와 성질

**이진 탐색 트리(BST, Binary Search Tree)**는 비어있을 수 있는 이진 트리이다. 비어있지 않은 BST는 다음을 만족한다:

1. 각 노드는 키(또는 값)를 가지며, **모든 키는 서로 다르다**
2. 모든 노드 $x$에 대해, $x$의 **왼쪽 서브트리**에 있는 모든 키는 $x$의 키보다 **작다**
3. 모든 노드 $x$에 대해, $x$의 **오른쪽 서브트리**에 있는 모든 키는 $x$의 키보다 **크다**
4. 루트의 왼쪽과 오른쪽 서브트리도 이진 탐색 트리이다

> **핵심:** BST 성질은 재귀적이다: 모든 서브트리가 그 자체로 유효한 BST이다. BST의 **중위 순회**는 키를 **오름차순**(정렬된 순서)으로 출력한다.

### 1.2 예시

```
    30            60
   /  \            \
  5    40          70
 / \              /  \
2   -            65   80
```

루트가 30인 트리(b)는 BST이다: 왼쪽 서브트리 키 {2, 5} < 30 < {40} 오른쪽 서브트리. 루트가 60인 트리(c)도 BST이다. 그러나 왼쪽 서브트리에 루트보다 큰 키가 포함된 트리는 BST가 **아니다**.

### 1.3 표현

BST의 원소 수와 형태는 삽입과 삭제에 따라 동적으로 변하므로, BST는 보통 **연결 표현**(각 노드가 `leftChild`, `element`, `rightChild` 포인터를 가짐)을 사용하여 표현된다.

## 2. BST 연산

### 2.1 탐색

BST에서 키 $k$를 탐색하려면:

1. **루트**에서 시작
2. 루트가 NULL이면 탐색 **실패** (키를 찾지 못함)
3. $k <$ 루트의 키이면 **왼쪽 서브트리**를 탐색
4. $k >$ 루트의 키이면 **오른쪽 서브트리**를 탐색
5. $k =$ 루트의 키이면 탐색 **성공**

```cpp
Node* search(Node* node, int key) {
    if (node == nullptr || node->key == key)
        return node;
    if (key < node->key)
        return search(node->left, key);
    else
        return search(node->right, key);
}
```

**시간 복잡도:** $O(h)$, 여기서 $h$는 트리의 높이.

### 2.2 삽입

키 $k$를 가진 새 원소를 삽입하려면:

1. BST에서 $k$를 **탐색**
2. 탐색이 **성공**하면(키가 이미 존재) **삽입하지 않음** (중복)
3. 탐색이 **실패**하면, 탐색이 종료된 지점에 새 원소를 삽입

```cpp
Node* insert(Node* node, int key) {
    if (node == nullptr)
        return new Node(key);
    if (key < node->key)
        node->left = insert(node->left, key);
    else if (key > node->key)
        node->right = insert(node->right, key);
    // key == node->key이면 아무것도 하지 않음 (중복 불허)
    return node;
}
```

**시간 복잡도:** $O(h)$ — 삽입 지점은 항상 리프 또는 null 포인터에 있다.

### 2.3 삭제

삭제는 삭제할 노드에 따라 **세 가지 경우**가 있다:

**경우 1: 리프 노드 (차수 0)**
- 부모의 포인터를 NULL로 설정하여 단순히 노드를 제거

**경우 2: 차수 1 노드 (자식 하나)**
- 노드를 유일한 자식으로 대체 (자식이 노드의 위치를 차지)

**경우 3: 차수 2 노드 (자식 둘)**
- 노드의 키를 다음 중 하나로 대체:
  - **왼쪽 서브트리의 가장 큰 키** (중위 전임자), 또는
  - **오른쪽 서브트리의 가장 작은 키** (중위 후임자)
- 그런 다음 전임자/후임자 노드를 삭제 (이 노드는 차수 0 또는 1이 보장됨)

**중위 전임자 찾기:** 왼쪽 자식으로 이동한 후, 오른쪽 자식 포인터가 NULL인 노드에 도달할 때까지 오른쪽 자식 포인터를 따라간다.

**중위 후임자 찾기:** 오른쪽 자식으로 이동한 후, 왼쪽 자식 포인터가 NULL인 노드에 도달할 때까지 왼쪽 자식 포인터를 따라간다.

**시간 복잡도:** 세 가지 경우 모두 $O(h)$.

> **핵심:** 경우 3이 가장 까다롭다. 대체 키(중위 전임자 또는 후임자)는 항상 차수 0 또는 1인 노드에 있으므로, 경우 3은 경우 1 또는 경우 2로 귀결된다.

## 3. BST 높이 분석

모든 BST 연산(탐색, 삽입, 삭제)은 시간 복잡도가 $O(h)$이며, 여기서 $h$는 트리의 높이이다.

### 3.1 최선의 경우

$n$개 노드를 가진 균형 BST의 높이는 $h = \lceil \log_2(n+1) \rceil$이며, $O(\log n)$ 연산을 제공한다.

이는 키가 균형 잡힌 순서로 삽입될 때 발생한다 (예: 중앙값을 먼저, 그 다음 부분 배열의 중앙값 등).

### 3.2 최악의 경우

$n$개 노드를 가진 퇴화(편향) BST의 높이는 $h = n$이며, $O(n)$ 연산을 제공한다.

이는 키가 **정렬된 순서**(오름차순 또는 내림차순)로 삽입될 때 발생하여 연결 리스트 구조가 된다.

| 시나리오 | 높이 | 연산 시간 |
|:---------|:-----|:---------|
| 최선의 경우 (균형) | $O(\log n)$ | $O(\log n)$ |
| 최악의 경우 (편향) | $O(n)$ | $O(n)$ |
| 평균적인 경우 (랜덤) | $O(\log n)$ | $O(\log n)$ |

> **핵심:** $O(\log n)$ 높이를 보장하려면, AVL 트리, Red-Black 트리, 또는 B-트리와 같은 자가 균형 BST를 사용한다.

## 4. 인덱스 이진 탐색 트리

### 4.1 정의

**인덱스 BST**는 각 노드에 `leftSize` 필드를 추가하며, 이는 왼쪽 서브트리의 원소 수 + 1 (즉, 서브트리 내에서 해당 노드의 **순위**)을 저장한다.

```
         8 [20]         <- leftSize=8, key=20
        /       \
    4 [10]     3 [40]
    /    \     /    \
 1[6]  1[15] 1[30]  0[42]
```

### 4.2 순위 기반 연산

`leftSize` 필드를 사용하면 다음을 수행할 수 있다:
- **순위로 찾기** ($k$번째로 작은 원소 얻기) — $O(h)$
- **키의 순위** — $O(h)$

이러한 연산은 인덱스가 없는 표준 BST에서는 효율적이지 않다.

## 5. 중복이 있는 BST

BST에서 중복 키를 허용하려면, 정의를 수정한다:
1. 성질 2에서 "작다"를 "작거나 같다"로 변경
2. 성질 3에서 "크다"를 "크거나 같다"로 변경

이를 통해 동일한 키 값을 가진 여러 노드가 허용된다.

## 요약

| 개념 | 핵심 |
|:-----|:-----|
| BST 정의 | 왼쪽 서브트리 키 < 루트 < 오른쪽 서브트리 키; 재귀적 |
| 중위 순회 | 키를 오름차순으로 출력 |
| 탐색 | $O(h)$ — 비교 후 왼쪽 또는 오른쪽으로 재귀 |
| 삽입 | $O(h)$ — 삽입 지점을 찾기 위해 탐색 |
| 리프 삭제 | 단순히 노드를 제거 |
| 차수 1 삭제 | 유일한 자식으로 대체 |
| 차수 2 삭제 | 중위 전임자 또는 후임자로 대체 |
| 최선 높이 | $O(\log n)$ — 균형 트리 |
| 최악 높이 | $O(n)$ — 퇴화 (정렬 삽입) |
| 인덱스 BST | `leftSize` 필드로 순위 기반 질의 $O(h)$ 가능 |

## 부록

### 다른 딕셔너리 구조와의 비교

| 구조 | 탐색 | 삽입 | 삭제 |
|:-----|:-----|:-----|:-----|
| 정렬 배열 | $O(\log n)$ | $O(n)$ | $O(n)$ |
| 스킵 리스트 | $O(\log n)$ 평균 | $O(\log n)$ 평균 | $O(\log n)$ 평균 |
| 해시 테이블 (체이닝) | $O(1)$ 평균 | $O(1)$ 평균 | $O(1)$ 평균 |
| BST | $O(\log n)$ 평균 | $O(\log n)$ 평균 | $O(\log n)$ 평균 |
| 균형 BST (AVL/RB) | $O(\log n)$ | $O(\log n)$ | $O(\log n)$ |

BST는 **순차 접근**(중위 순회)과 **순위 기반 질의**에 특히 이상적이며, 해시 테이블은 이를 효율적으로 지원하지 못한다.

### 면접 팁

- 세 가지 삭제 경우를 모두 알고 예시를 추적할 수 있어야 함
- BST가 퇴화되는 경우와 균형 BST가 이를 방지하는 방법을 이해할 것
- BST의 중위 순회는 BST 성질을 검증하는 고전적 방법
