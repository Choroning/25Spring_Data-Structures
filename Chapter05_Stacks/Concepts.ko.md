## 1. 스택 ADT

### 1.1 정의

**스택(Stack)** 은 **탑(top)** 이라 불리는 한쪽 끝에서만 삽입과 삭제가 이루어지는 선형 리스트이다.

- **LIFO** (Last In, First Out) 원칙
- 푸시다운 리스트(pushdown list)라고도 함

### 1.2 핵심 연산

| 연산 | 설명 | 복잡도 |
|------|------|--------|
| `push(x)` | 원소 $x$를 탑에 삽입 | $O(1)$ |
| `pop()` | 탑 원소를 제거하고 반환 | $O(1)$ |
| `top()` | 탑 원소를 제거하지 않고 반환 | $O(1)$ |
| `empty()` | 스택이 비어 있는지 확인 | $O(1)$ |
| `size()` | 원소의 개수를 반환 | $O(1)$ |

> **핵심 포인트:** 모든 스택 연산은 $O(1)$이다 — 이것이 스택을 효율적으로 만드는 이유이다.

## 2. 배열 기반 스택

### 2.1 구현

배열과 탑을 추적하는 변수 `topIndex`를 사용한다:
- `topIndex = -1`이면 스택이 비어 있음
- `push(x)`: `topIndex`를 증가시키고 `array[topIndex]`에 저장
- `pop()`: `array[topIndex]`를 반환하고 `topIndex`를 감소

### 2.2 동적 크기 조정

배열이 가득 찼을 때 크기를 두 배로 늘린다 (분할 상환 $O(1)$ push).

### 2.3 장단점

- **장점**: 캐시 친화적, 간단한 구현
- **단점**: 용량이 실제 크기보다 훨씬 큰 경우 공간 낭비 가능

## 3. 연결 스택

### 3.1 구현

단일 연결 리스트를 사용하며, 스택의 탑은 머리(head)이다:
- `push(x)`: 머리에 새 노드 삽입 — $O(1)$
- `pop()`: 머리 노드 제거 — $O(1)$

### 3.2 장단점

- **장점**: 공간 낭비 없음, 진정한 동적 구조
- **단점**: 원소당 추가 포인터 오버헤드, 불량한 캐시 지역성

## 4. 스택의 응용

### 4.1 괄호 매칭

괄호 `()`, `[]`, `{}`가 올바르게 중첩되어 있는지 확인한다:

```cpp
bool isBalanced(const std::string& expr) {
    std::stack<char> s;
    for (char c : expr) {
        if (c == '(' || c == '[' || c == '{') s.push(c);
        else if (c == ')' || c == ']' || c == '}') {
            if (s.empty()) return false;
            char top = s.top(); s.pop();
            if ((c == ')' && top != '(') ||
                (c == ']' && top != '[') ||
                (c == '}' && top != '{'))
                return false;
        }
    }
    return s.empty();
}
```

### 4.2 함수 호출 스택

런타임 시스템은 함수 호출을 관리하기 위해 스택을 사용한다:
- 호출 시 활성 레코드를 push
- 반환 시 pop
- 재귀를 자연스럽게 지원

### 4.3 실행 취소 연산

텍스트 편집기, 브라우저 및 많은 애플리케이션이 실행 취소 기능을 위해 스택을 사용한다.

### 4.4 하노이의 탑

스택을 사용하여 우아하게 해결되는 고전적인 재귀 문제:

$$T(n) = 2T(n-1) + 1 = 2^n - 1 \text{ 이동}$$

## 5. 수식 평가

### 5.1 중위, 전위, 후위 표기법

| 유형 | 예시 | 설명 |
|------|------|------|
| 중위(Infix) | `A + B * C` | 연산자가 피연산자 사이에 위치 |
| 전위(Prefix, Polish) | `+ A * B C` | 연산자가 피연산자 앞에 위치 |
| 후위(Postfix, Reverse Polish) | `A B C * +` | 연산자가 피연산자 뒤에 위치 |

### 5.2 중위에서 후위로 변환

연산자용 스택을 사용하는 알고리즘:

1. 왼쪽에서 오른쪽으로 스캔
2. 피연산자이면 출력
3. `(`이면 스택에 push
4. `)`이면 `(`를 만날 때까지 pop하며 출력
5. 연산자이면, 우선순위가 높거나 같은 연산자를 pop하여 출력한 후 push

### 5.3 후위 수식 평가

1. 왼쪽에서 오른쪽으로 스캔
2. 피연산자이면 스택에 push
3. 연산자이면 두 피연산자를 pop하고, 연산자를 적용한 후 결과를 push

## 요약

| 개념 | 핵심 포인트 |
|------|-------------|
| 스택 | LIFO 자료구조; 모든 연산 $O(1)$ |
| 배열 기반 | `topIndex` 사용; 크기 조정이 필요할 수 있음 |
| 연결 | 연결 리스트의 머리를 탑으로 사용; 용량 제한 없음 |
| 괄호 매칭 | 여는 괄호 push, 닫는 괄호에서 pop |
| 수식 평가 | 중위를 후위로 변환 후 평가 |
| 재귀 | 함수 호출 스택이 암묵적으로 스택임 |

## 부록

### 스택 vs. 큐

| 특성 | 스택 | 큐 |
|------|------|-----|
| 순서 | LIFO | FIFO |
| 삽입 | push (탑) | enqueue (후단) |
| 삭제 | pop (탑) | dequeue (전단) |
| 활용 사례 | 실행 취소, 재귀, DFS | 스케줄링, BFS |
