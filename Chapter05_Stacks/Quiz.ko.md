## Q1. 스택 연산 추적

**문제:** 초기에 비어 있는 스택에 다음 연산을 수행한 후의 상태를 추적하라: `push(5)`, `push(3)`, `pop()`, `push(7)`, `push(2)`, `pop()`, `push(8)`. 탑 원소는 무엇인가?

**정답:** 탑 원소는 8이다. 바닥에서 탑 순서로의 스택: [5, 7, 8].

**해설:** `push(5)` 후: [5]. `push(3)` 후: [5, 3]. `pop()` 후: [5] (3 제거). `push(7)` 후: [5, 7]. `push(2)` 후: [5, 7, 2]. `pop()` 후: [5, 7] (2 제거). `push(8)` 후: [5, 7, 8].

## Q2. 중위에서 후위로 변환

**문제:** 중위 수식 `A + B * C - D / E`를 후위 표기법으로 변환하라.

**정답:** `A B C * + D E / -`

**해설:** 연산자 우선순위에 따라 (* 와 / 가 + 와 - 보다 우선): 먼저 `B * C`가 `B C *`가 된다. 그 다음 `A + (B C *)`가 `A B C * +`가 된다. 그 다음 `D / E`가 `D E /`가 된다. 마지막으로 `(A B C * +) - (D E /)`가 `A B C * + D E / -`가 된다.

## Q3. 후위 수식 평가

**문제:** 후위 수식 `6 2 3 + - 3 8 2 / + * 2 ^ 3 +`을 평가하라.

**정답:** 52.

**해설:** 단계별: `2 3 +` = 5; `6 5 -` = 1; `8 2 /` = 4; `3 4 +` = 7; `1 7 *` = 7; `7 2 ^` = 49; `49 3 +` = 52.

## Q4. 괄호 매칭

**문제:** 다음 수식의 괄호가 균형을 이루는지 판별하라: (a) `{[()]}` (b) `{[(])}` (c) `((()))`

**정답:** (a) 균형. (b) 불균형. (c) 균형.

**해설:** (a) 각 닫는 괄호가 가장 최근의 매칭되지 않은 같은 유형의 여는 괄호와 일치한다. (b) `]`가 스택에서 `[` 대신 `(`를 만나므로 불일치. (c) 모든 괄호가 올바르게 매칭된다.

## Q5. 배열 vs. 연결 스택

**문제:** 10개에서 1000개 사이로 원소 수가 변동하는 스택에 대해, 배열 기반 스택과 연결 스택의 메모리 사용량을 비교하라.

**정답:** 연결 스택은 현재 원소 수에 비례하는 메모리에 포인터 오버헤드를 더한 만큼 사용한다. 배열 기반 스택은 현재 원소가 10개뿐이더라도 1000개 이상의 원소를 위한 용량을 할당할 수 있다. 크기가 변동하는 경우, 연결 스택이 전체 메모리 낭비가 적지만, 각 원소가 포인터로 인해 더 많은 메모리를 사용한다.

**해설:** 스택이 1000까지 성장했다가 10으로 줄어들면, 배열 기반 스택은 여전히 1000개(또는 배가 후 1024개)분의 메모리를 보유한다. 연결 스택은 pop될 때 노드를 해제한다. 그러나 각 연결 노드는 포인터를 위한 추가 바이트(64비트 시스템에서 8바이트)를 사용한다.
