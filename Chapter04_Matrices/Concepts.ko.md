## 1. 배열과 행렬

### 1.1 다차원 배열

**이차원 배열**(행렬) `A[m][n]`은 $m$개의 행과 $n$개의 열로 구성된 $m \times n$개의 원소를 저장한다.

### 1.2 메모리 매핑

메모리는 선형이므로, 2차원 배열은 1차원으로 매핑되어야 한다. 매핑 함수는 원소가 어떻게 배치되는지를 결정한다.

차원이 $m \times n$인 2차원 배열 `A[r][c]`에 대해:

**행 우선 순서**: $\text{location}(A[i][j]) = \text{base} + (i \cdot n + j) \cdot \text{sizeof}(T)$

**열 우선 순서**: $\text{location}(A[i][j]) = \text{base} + (j \cdot m + i) \cdot \text{sizeof}(T)$

## 2. 행 우선 순서와 열 우선 순서

### 2.1 행 우선 순서 (C/C++ 기본값)

각 행의 원소가 연속적으로 저장된다:

```
A[0][0], A[0][1], ..., A[0][n-1], A[1][0], A[1][1], ..., A[m-1][n-1]
```

### 2.2 열 우선 순서 (Fortran, MATLAB)

각 열의 원소가 연속적으로 저장된다:

```
A[0][0], A[1][0], ..., A[m-1][0], A[0][1], A[1][1], ..., A[m-1][n-1]
```

> **핵심 포인트:** 행 우선 순서는 행 방향 순회 시 캐시 효율이 좋고, 열 우선 순서는 열 방향 순회 시 캐시 효율이 좋다.

## 3. 특수 행렬

### 3.1 대각 행렬

주 대각선의 원소만 0이 아닌 행렬: $i \neq j$일 때 $A[i][j] = 0$.

저장: $n \times n$ 행렬에 대해 $n$개의 원소만 필요.

$$\text{map}(i) = i \quad \text{(}A[i][i]\text{를 위치 }i\text{에 저장)}$$

### 3.2 삼중 대각 행렬

주 대각선과 인접한 두 대각선에만 0이 아닌 원소가 존재한다.

저장: $n \times n$ 행렬에 대해 $3n - 2$개의 원소.

### 3.3 삼각 행렬

**하삼각 행렬**: $j > i$일 때 $A[i][j] = 0$

원소의 수: $\frac{n(n+1)}{2}$

매핑: $\text{map}(i,j) = \frac{i(i+1)}{2} + j$ (행 우선 하삼각)

**상삼각 행렬**: $j < i$일 때 $A[i][j] = 0$

### 3.4 대칭 행렬

모든 $i, j$에 대해 $A[i][j] = A[j][i]$. 하삼각(또는 상삼각) 부분만 저장하면 된다: $\frac{n(n+1)}{2}$개의 원소.

## 4. 희소 행렬

### 4.1 정의

행렬의 원소 중 대부분이 0인 경우 **희소(sparse)** 행렬이라 한다. 일반적으로, $n \times n$ 행렬에서 0이 아닌 원소의 수가 $O(n^2)$이 아닌 $O(n)$ 또는 $O(n \log n)$이다.

### 4.2 삼중쌍(Triples) 배열 표현

각 0이 아닌 원소를 삼중쌍 $(i, j, \text{value})$으로 저장한다:

```cpp
struct MatrixTerm {
    int row;
    int col;
    int value;
};
```

0이 아닌 원소가 $t$개인 행렬에 대해:
- 공간: $O(m \times n)$ 대신 $O(t)$
- 접근: 원소를 찾는 데 $O(t)$ (선형 탐색)

### 4.3 희소 행렬의 전치

삼중쌍 표현의 경우, 행과 열 인덱스를 교환한 후 새로운 행 순서로 정렬하여 전치한다.

**단순 접근법**: $O(t \cdot n)$ (여기서 $t$ = 항의 수, $n$ = 열의 수)
**빠른 전치**: 열 개수 배열을 사용하여 $O(t + n)$

## 5. 희소 행렬의 연결 표현

### 5.1 직교 리스트

각 0이 아닌 원소는 다음을 포함하는 노드이다:
- `row`, `col`, `value`
- `nextInRow` (같은 행의 다음 0이 아닌 원소를 가리키는 포인터)
- `nextInCol` (같은 열의 다음 0이 아닌 원소를 가리키는 포인터)

추가로 행과 열의 헤드 포인터 배열이 있다.

### 5.2 장점

- 0이 아닌 원소의 효율적인 삽입 및 삭제
- 덧셈 등의 연산에 자연스러운 표현

## 6. 행렬 연산

### 6.1 희소 행렬 덧셈

삼중쌍 형태의 두 희소 행렬에 대해:
- 정렬된 삼중쌍 리스트를 병합
- 겹치는 위치는 값을 더함

시간 복잡도: $O(t_1 + t_2)$

### 6.2 희소 행렬 곱셈

시간 복잡도는 표현 방식에 따라 다르지만, 행렬이 충분히 희소한 경우 밀집 곱셈보다 일반적으로 더 효율적이다.

## 부록

### 희소 표현을 사용할 시기

밀도 $d = \frac{\text{0이 아닌 원소 수}}{m \times n}$가 대략 0.3(30%) 미만일 때 희소 표현을 사용한다. 이 임계값 이상에서는 희소 구조의 오버헤드가 절약분을 초과할 수 있다.

### 희소 행렬의 응용

- 유한 요소 해석
- 네트워크 인접 행렬 (소셜 그래프, 웹 링크 그래프)
- 자연어 처리 (단어-문서 행렬)
- 과학 계산 (PDE 이산화)
